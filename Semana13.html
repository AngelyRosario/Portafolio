Semana 13

Descripción

Esta semana nos enfocamos en el desarrollo de aplicaciones backend utilizando Jakarta Server Pages (JSP). Explora cómo JSP y servlets pueden ser utilizados para manejar solicitudes web y generar respuestas dinámicas, integrando patrones de diseño como la arquitectura MVC y el uso del Spring Framework para una gestión más eficiente.

Tópicos Aprendidos

Definiciones

Arquitectura MVC es un patrón de diseño que separa el desarrollo en tres capas: Modelo (datos), Vista (interfaz de usuario) y Controlador (lógica de negocio). Este enfoque organiza el flujo de trabajo, comenzando desde la vista donde se realizan las solicitudes y se procesan en el controlador. Su aplicación inicial incluía la protección de directorios públicos mediante archivos htaccess y htpasswd, lo que llevó a la creación de sistemas de rutas controladas.

Procedimientos

  1. Desarrollo de Servlets y JSPs: Implementé servlets y JSPs para manejar solicitudes web y generar respuestas dinámicas. Incluye la creación de un servlet para mostrar información de la solicitud y un formulario JSP para capturar y mostrar datos de usuarios.
  2. Integración del Spring Framework: Introduje el Spring Framework para mejorar la gestión de dependencias y proporcionar un modelo completo para el desarrollo de aplicaciones empresariales modernas, integrando patrones de diseño como la Inyección de Dependencia.
  3. Implementación y Pruebas: Realicé la implementación en un entorno con Tomcat 11 y JDK-24, asegurando la compatibilidad y el buen funcionamiento de las aplicaciones desarrolladas.

Resultados

El resultado fue una serie de aplicaciones web funcionales que utilizan servlets y JSP para procesar solicitudes y generar contenido dinámico. La integración del Spring Framework facilitó la gestión de dependencias y mejoró el desarrollo de aplicaciones empresariales. La implementación en un entorno actualizado garantizó la compatibilidad y eficiencia de las aplicaciones.

Reflexión Metacognitiva

Reflexioné sobre la importancia de aplicar patrones de diseño como MVC y buenas prácticas en el desarrollo de aplicaciones web empresariales. La integración de herramientas como el Spring Framework demuestra cómo la aplicación de patrones y tecnologías adecuadas puede mejorar significativamente la calidad y la eficiencia del desarrollo de software.

¿Qué aprendí?

Aprendí a utilizar servlets y JSP para manejar solicitudes y generar contenido dinámico, y a integrar el Spring Framework para optimizar el desarrollo de aplicaciones Java, aplicando la arquitectura MVC y patrones de diseño avanzados.

¿Cómo aprendí?

Mi aprendizaje se dio a través de la práctica en proyectos y la realización de ejercicios enfocados en la arquitectura MVC, servlets, JSP y el Spring Framework. Cada ejercicio me permitió aplicar y consolidar los conceptos aprendidos en el desarrollo de aplicaciones web.
